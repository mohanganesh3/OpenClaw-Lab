## Problem
Evaluating machine learning agents on machine learning engineering tasks through a structured competition framework

## Core Claim
MLE-bench provides a standardized benchmark for assessing the capabilities of ML agents in performing ML engineering tasks

## Key Contributions
- A comprehensive evaluation framework for ML agents on ML engineering tasks
- A specific metric (Log of Mean Absolute Error) for evaluating predictions across multiple scalar coupling types
- A structured competition setup with defined data, evaluation protocols, and timeline

## Method
The paper describes a competition framework where ML agents predict scalar coupling constants between atom pairs in molecules, evaluated using a log-MAE metric averaged across coupling types
- Components: Data preprocessing from molecular structure files (XYZ and CSV formats), Feature engineering from molecular properties (dipole moments, shielding tensors, charges, etc.), Structured evaluation with log-MAE metric and specific submission format
- Key innovation: The single most important innovation is the log-MAE evaluation metric that provides equal weight to improvements across different scalar coupling types, ensuring fair comparison
- Assumptions: Molecular structure data contains sufficient information for predicting scalar coupling constants; The provided supplementary data (dipole moments, shielding tensors, etc.) are relevant features; The log transformation of MAE appropriately handles the scale of coupling constants

## Experimental Setup
- Datasets: scalar coupling constant prediction dataset (train.csv, test.csv, structures.zip, structures.csv, dipole_moments.csv, magnetic_shielding_tensors.csv, mulliken_charges.csv, potential_energy.csv, scalar_coupling_contributions.csv)
- Baselines: o1-preview, gpt-4o-2024-08-06, llama-3.1-405b-instruct, claude-3-5-sonnet-20240620, gpt-4o (MLAB), gpt-4o (OpenHands)
- Metrics: Log Mean Absolute Error (Log MAE)
