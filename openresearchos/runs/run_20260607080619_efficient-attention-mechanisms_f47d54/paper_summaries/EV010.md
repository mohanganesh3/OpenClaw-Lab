## Problem
Language model agents lack specialized interfaces for complex software engineering tasks, unlike human engineers who benefit from IDEs.

## Core Claim
LM agents require specially-built interfaces to effectively perform software engineering tasks, similar to how humans need IDEs.

## Key Contributions
- SWE-agent system that enables LM agents to autonomously use computers for software engineering tasks
- Custom agent-computer interface (ACI) that enhances LM agents' ability to create/edit code, navigate repositories, and execute tests

## Method
SWE-agent facilitates LM agents to autonomously use computers through an ACI that includes specific commands for file manipulation and navigation, with structured feedback from the computer environment.
- Components: LM-friendly commands for file operations and navigation, LM-friendly environment feedback format, Agent-computer interface connecting LM agent to computer
- Key innovation: The ACI design that bridges LM capabilities with software engineering needs through carefully crafted commands and feedback mechanisms.
- Assumptions: LM agents can benefit from structured interfaces like humans do; Interface design significantly impacts agent performance

## Experimental Setup
- Datasets: SWE-bench Lite, SWE-bench (full)
- Baselines: SWE-agent w/ GPT-4, SWE-agent w/ Claude 3 Opus
- Metrics: Resolution rate, Turn distribution, Total steps, Cost (USD)
- Compute: null

## Results
Headline: SWE-agent with GPT-4 resolved 60% of tasks within 25 turns on SWE-bench (full), while Claude 3 Opus resolved 50% within 20 turns on SWE-bench Lite
