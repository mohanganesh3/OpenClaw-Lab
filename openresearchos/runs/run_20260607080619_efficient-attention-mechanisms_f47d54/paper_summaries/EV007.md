# EV007: Self-Refine: Iterative Refinement with Self-Feedback

Year: 2023 | Relation: ? | Repro: low | Source: pymupdf_full (54p, 21 tables)
URL: https://arxiv.org/abs/2303.17651 | Code: no

## Problem
Large language models often produce suboptimal outputs on their first attempt, especially for tasks with complex requirements or poorly defined objectives.

## Core Claim
SELF-REFINE can significantly improve LLM outputs through iterative self-feedback without requiring additional training, data, or reinforcement learning.

## Key Contributions
- A novel approach for iterative refinement using self-feedback within a single LLM
- Demonstrated effectiveness across 7 diverse tasks with ~20% absolute improvement in task performance

## Method
SELF-REFINE generates an initial output using an LLM, then has the same LLM provide feedback for its output and use it to refine itself iteratively.
- Components: Initial output generation, Self-feedback generation, Iterative refinement
- Key innovation: Using a single LLM to both generate feedback and refine its own outputs without any additional training or data.
- Assumptions: The LLM can provide useful feedback for its own outputs; Iterative refinement will lead to improvements in output quality

## Experimental Setup
- Datasets: CommonGen-Hard, Acronym Generation (250 acronyms), CodeNet (300 examples)
- Baselines: GPT-3.5, ChatGPT, GPT-4
- Metrics: GPT-4-based preference evaluation, percentage of programs optimized, solve rate, coverage percentage, meaningful variable ratio, comment per line, function units
- Compute: null

## Results
Headline: GPT-4 with SELF-REFINE on Constrained Generation: 45.0% (from 15.0% base)

| Method | Dataset | Metric | Value |
|---|---|---|---|
| GPT-3.5 + SELF-REFINE | Sentiment Reversal | GPT-4-based preference | 30.4% |
| GPT-3.5 | Sentiment Reversal | GPT-4-based preference | 8.8% |
| GPT-4 + SELF-REFINE | Constrained Gen. | coverage percentage | 45.0% |
| GPT-4 | Constrained Gen. | coverage percentage | 15.0% |
| ChatGPT + SELF-REFINE | Code Readability | GPT-4-based preference | 63.1% |
| ChatGPT | Code Readability | GPT-4-based preference | 27.7% |

- Ablations: Error analysis shows model robustness to bad feedback (60%) and issues like ignoring feedback (25%)
- Improvement over baseline: Varies by task; e.g., GPT-4 Constrained Gen. improved by 30.0 percentage points
