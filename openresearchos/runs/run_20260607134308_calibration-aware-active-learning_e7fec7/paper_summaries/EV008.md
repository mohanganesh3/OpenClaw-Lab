## Problem
Creating embodied agents that can continuously explore, plan, and develop new skills in open-ended worlds without human intervention.

## Core Claim
VOYAGER is the first LLM-powered embodied lifelong learning agent that can continuously explore the world, acquire diverse skills, and make novel discoveries without human intervention in Minecraft.

## Key Contributions
- An automatic curriculum that maximizes exploration
- An ever-growing skill library of executable code for storing and retrieving complex behaviors
- A new iterative prompting mechanism that incorporates environment feedback, execution errors, and self-verification for program improvement

## Method
VOYAGER interacts with GPT-4 via blackbox queries to generate executable code for Minecraft actions, consisting of three components: an automatic curriculum proposing tasks based on exploration progress, a skill library storing and retrieving complex behaviors, and an iterative prompting mechanism refining programs using environment feedback, execution errors, and self-verification.
- Components: Automatic curriculum for open-ended exploration, Skill library for storing and retrieving complex behaviors, Iterative prompting mechanism using code as action space
- Key innovation: The iterative prompting mechanism that incorporates environment feedback, execution errors, and self-verification for program improvement, with self-verification being most critical as it determines when to move to new tasks or reattempt unsuccessful ones.
- Assumptions: GPT-4 can generate high-quality executable code for Minecraft actions; The Minecraft environment provides sufficient feedback signals for learning; The skill library can effectively store and retrieve complex behaviors; The automatic curriculum can propose achievable tasks that drive exploration

- Threats to validity: Domain mismatch: Minecraft vs. medical tabular data; Generalization concerns: Skills learned in Minecraft may not transfer to medical data; Potential overfitting to the Minecraft environment; Limited evaluation of performance on tasks outside Minecraft domain
- Open questions: How can the approach be adapted for non-embodied domains like medical tabular data?; What are the computational costs and resource requirements?; How does the agent handle noisy or uncertain information?; Can the skills learned in Minecraft be transferred to other domains?
