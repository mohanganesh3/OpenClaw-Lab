## Problem
Lack of comprehensive quantitative evaluation for LLMs acting as agents in interactive environments.

## Core Claim
While top commercial LLMs show strong agent capabilities, significant performance gaps exist with open-source models, primarily due to poor long-term reasoning, decision-making, and instruction following abilities.

## Key Contributions
- AGENTBENCH: A multi-dimensional benchmark with 8 distinct environments for evaluating LLM-as-Agent capabilities
- Extensive evaluation of 29 API-based and open-source LLMs revealing performance disparities
- Identification of key failure modes and obstacles in LLM agent development

## Method
AGENTBENCH creates 8 simulated interactive environments (Operating System, Database, Knowledge, Graph, Digital Card Game, Lateral Thinking Puzzle, House-holding, Web) where LLMs perform agent tasks, with performance measured through automated evaluation.
- Components: Eight distinct interactive environments covering different agent challenges, Automated evaluation framework measuring task completion success, Comparative analysis across API-based and open-source LLMs
- Key innovation: Multi-environment design capturing diverse agent capabilities (reasoning, decision-making, instruction following) beyond traditional benchmarks, revealing critical failure modes in LLM agents.
- Assumptions: Simulated environments adequately represent real-world agent challenges; LLM outputs can be objectively evaluated through automated metrics; Performance gaps reflect fundamental limitations rather than evaluation artifacts

## Experimental Setup
- Datasets: FREEBASE-based dataset with 1,663 questions from GrailQA, ComplexWebQuestions, and GraphQuestions, Text-to-SQL dataset with INSERT, UPDATE, and SELECT operations
- Baselines: gpt-4, gpt-3.5-turbo
- Metrics: F1 score, Exact Match, Executability
