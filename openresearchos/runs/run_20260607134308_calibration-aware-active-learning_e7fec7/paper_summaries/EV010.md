## Problem
Existing interfaces for LM agents (e.g., Linux shell) are not optimized for complex software engineering tasks, limiting their performance.

## Core Claim
LM agents significantly benefit from specially-designed interfaces like SWE-agent's agent-computer interface (ACI) for software engineering tasks.

## Key Contributions
- Introduction of SWE-agent with a custom agent-computer interface (ACI) for LM agents
- Achieved state-of-the-art performance on SWE-bench (12.5% pass@1) and HumanEvalFix (87.7% pass@1)

## Method
SWE-agent uses a custom agent-computer interface (ACI) with LM-friendly commands and structured feedback to enable autonomous software engineering tasks.
- Components: Filesystem navigation and viewing capabilities, LM-friendly commands for searching and editing files, Structured environment feedback
- Key innovation: The agent-computer interface (ACI) designed with LM-friendly commands and feedback formats that align with LM agents' capabilities.
- Assumptions: LM agents can effectively utilize structured interfaces; The ACI's design aligns with LM agents' capabilities

## Experimental Setup
- Datasets: SWE-bench Lite dataset, SWE-bench test set (286 trajectories), SWE-bench Lite split (35 trajectories)
- Baselines: SWE-agent with GPT-4 on full SWE-bench test set, SWE-agent with Claude 3 Opus on Lite split
- Metrics: % Resolved performance, Distribution of turns needed, Total steps distribution, Cost distribution (USD), Action pattern frequencies
