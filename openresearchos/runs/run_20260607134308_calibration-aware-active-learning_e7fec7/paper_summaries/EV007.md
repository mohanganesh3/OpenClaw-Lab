# EV007: Self-Refine: Iterative Refinement with Self-Feedback

Year: 2023 | Relation: method_source | Repro: medium | Source: pymupdf_full (54p, 21 tables)
URL: https://arxiv.org/abs/2303.17651 | Code: yes

## Problem
Large language models often fail to generate optimal outputs on their first attempt, especially for tasks with multifaceted objectives or hard-to-define goals.

## Core Claim
SELF-REFINE can improve LLM outputs through iterative self-feedback without requiring supervised training data, additional training, or reinforcement learning.

## Key Contributions
- Introduction of SELF-REFINE, an approach for improving initial LLM outputs through iterative feedback and refinement
- Demonstration that SELF-REFINE works across 7 diverse tasks using state-of-the-art LLMs (GPT-3.5 and GPT-4), with outputs preferred by humans and automatic metrics over conventional one-step generation

## Method
SELF-REFINE generates an initial output using an LLM, then has the same LLM provide feedback for its output and use it to refine itself iteratively. This process repeats for multiple iterations to improve output quality.
- Components: Initial generation using an LLM, Self-feedback generation by the same LLM, Refinement of the output based on the feedback, Iterative repetition of self-feedback and refinement steps
- Key innovation: Using the same LLM as both the generator and the feedback provider, enabling iterative refinement without requiring any additional training data, supervised training, or reinforcement learning.
- Assumptions: The LLM can provide meaningful feedback on its own outputs; The LLM can effectively use the feedback to improve its outputs; The approach works across diverse tasks and domains

## Experimental Setup
- Datasets: CommonGen-Hard (extension of CommonGen), Acronym Generation (250 acronyms), CodeNet competitive programming (300 examples), Human annotator subset (60 examples)
- Baselines: Base GPT-3.5, Base ChatGPT, Base GPT-4, Human annotators (code readability)
- Metrics: solve rate, percentage of programs optimized, GPT-4-based preference evaluation, meaningful variable ratio, comment per line, function units, coverage percentage
- Compute: not specified

## Critical Reading
- Stated limitations: —
- Unstated weaknesses: Approach may struggle with structured tabular data; No explicit handling of noisy labels or calibration issues; May not be optimized for small datasets where sample efficiency is crucial; Potential for feedback loops that reinforce initial mistakes
- Threats to validity: May not generalize well to medical domains where precision is critical; No guarantee that the refinement process improves calibration; May not address the specific challenges of active learning in noisy label scenarios; Small medical datasets may not have sufficient diversity for effective self-feedback
- Open questions: How does the self-refinement process affect model calibration?; Can the approach be adapted for tabular data specifically?; How effective is it when dealing with noisy labels?; What is the optimal number of refinement iterations?
