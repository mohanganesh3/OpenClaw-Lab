## Problem
Theoretical properties of stacking (stacked generalization) methods are poorly understood despite their widespread practical use in machine learning ensembles.

## Core Claim
Selecting the best stacking method from a family using cross-validated performance achieves performance close to the oracle best, with explicit theoretical guarantees.

## Key Contributions
- Proved that cross-validated selection of stacking methods provides performance within (1+2δ) of the oracle best plus vanishing error terms (Theorem 4.1)
- Extended Van der Laan et al. (2007) results to handle ordered datasets, infinite families, and non-constant stacking functions
- Proposed a family of stacking methods for probabilistic time series forecasting with controlled weight sensitivity

## Method
Proved an oracle inequality (Theorem 5.1) for general algorithms using Bernstein numbers and covering numbers, then applied it to stacking by defining stacking algorithms as compositions of base learners with weight functions Aα(Z)
- Components: Bernstein number analysis of function class F = {Lα(D0)}, Lipschitz continuity assumption on α → Lα(D0) mapping, Cross-validation set Ŵα = {˜α | θ̂α outperforms θ˜α on validation}
- Key innovation: Recognition that general oracle inequalities (Theorem 5.1) directly apply to stacking without discretization, enabling performance bounds that depend only on log(2+m) rather than discretization size
- Assumptions: J is bounded subset of finite-dimensional Euclidean space; For every D0, α → Lα(D0) is Lipschitz with constant ℓ; For p<2: inf_{f∈F} ∫fdP ≥ c > 0; ε_{n1} = n1^{-1/2 - ε1} for any ε1 > 0
