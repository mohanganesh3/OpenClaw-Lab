# EV007: Self-Refine: Iterative Refinement with Self-Feedback

## Problem
Large language models often fail to generate optimal outputs on their first attempt, especially for tasks with complex requirements or multifaceted objectives.

## Core Claim
SELF-REFINE demonstrates that state-of-the-art LLMs can be significantly improved at test-time through iterative self-feedback and refinement without requiring additional training or supervision.

## Key Contributions
- Introduces SELF-REFINE methodology for iterative refinement using self-feedback
- Demonstrates effectiveness across 7 diverse tasks with ~20% absolute improvement in task performance

## Method
SELF-REFINE generates initial output with an LLM, then iteratively uses the same LLM to provide feedback on its output and refine it, without requiring any additional training or external supervision.
- Components: Initial output generation using LLM, Self-feedback generation by the same LLM, Iterative refinement based on feedback
- Key innovation: The use of the same LLM as both the feedback provider and refiner, enabling iterative improvement without additional training or external supervision.
- Assumptions: The LLM can provide meaningful feedback on its own outputs; Iterative refinement will improve output quality; The approach works across diverse tasks without task-specific tuning

## Experimental Setup
- Datasets: CodeNet (300 examples), CommonGen-Hard (extension of CommonGen), Acronym dataset (250 acronyms)
- Baselines: Base GPT-3.5, Base ChatGPT, Base GPT-4
- Metrics: Solve rate, Percentage of programs optimized, GPT-4-based preference evaluation, Coverage percentage, Meaningful Variable Ratio, Comment Per Line, Function Units
- Compute: Not specified
