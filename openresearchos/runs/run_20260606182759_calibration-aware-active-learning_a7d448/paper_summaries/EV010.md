## Problem
Language model agents lack specially-built interfaces for complex software engineering tasks, limiting their effectiveness compared to human engineers who benefit from sophisticated applications like IDEs.

## Core Claim
Custom agent-computer interfaces can significantly enhance language model agents' ability to perform software engineering tasks by providing LM-friendly commands and feedback formats.

## Key Contributions
- Introduction of SWE-agent: a system facilitating LM agents to autonomously use computers for software engineering tasks
- Design of custom agent-computer interface (ACI) that enhances code creation, repository navigation, and test execution
- Achieving state-of-the-art performance on SWE-bench (12.5% pass@1) and HumanEvalFix (87.7%) benchmarks

## Method
SWE-agent implements a custom agent-computer interface (ACI) that provides LM-friendly commands and environment feedback, enabling language model agents to interact with computers for software engineering tasks including file manipulation, repository navigation, and program execution.
- Components: Agent-Computer Interface (ACI) with LM-friendly commands, Filesystem interaction capabilities for code creation and editing, Repository navigation and file management tools, Terminal interface for executing tests and programs
- Key innovation: The agent-computer interface (ACI) that translates complex software engineering operations into LM-friendly commands and provides structured feedback, effectively bridging the gap between language model capabilities and software engineering requirements.
- Assumptions: Language model agents can effectively utilize computers when provided with appropriate interfaces; Interface design significantly impacts agent performance beyond just the underlying LM capabilities; LM agents represent a new category of end users with distinct needs from human users
