## Problem
Existing unconventional computing approaches require problem transformations (embedding, sparsification, quadratisation) that degrade solution quality for higher-order, densely connected, or reconfigurable combinatorial optimization problems.

## Core Claim
A virtually connected probabilistic computer (VCPC) using photonic quantum random number generators outperforms digital annealing units by orders of magnitude on Erdős–Rényi graph spin-glasses without requiring problem transformations.

## Key Contributions
- Demonstration of VCPC's capability to solve higher-order problems (hypergraph hitting set) directly without quadratisation
- Implementation of greedy graph coloring for hardware parallelisation enabling favourable scaling
- Quantitative comparison showing quadratisation/sparsification would increase problem size by 10-25x for k=5/10 hypergraphs

## Method
Uses high-speed photonic quantum random number generators to create a probabilistic computing architecture with virtual connections between p-bits. Solves higher-order problems via simulated annealing (PC-SA) with greedy graph coloring for parallelization.
- Components: Photonic quantum random number generators for probabilistic p-bit states, Virtual connectivity layer enabling arbitrary problem topologies, Greedy hypergraph coloring for parallel p-bit updates, Energy function formulation for hypergraph hitting set (Eqn. 13)
- Key innovation: Virtual hardware connections allowing direct mapping of arbitrary combinatorial optimization problems to hardware without transformation, preserving problem structure and solution quality.
- Assumptions: Simulations accurately model VCPC hardware behavior; Photonic RNGs can generate high-speed random numbers; Greedy coloring provides sufficient parallelization for desired scaling

## Experimental Setup
- Datasets: burma14 TSP instance, Erdős–Rényi graphs (N=100, 250, 500, 750, 1024), Spin-glass instances
- Baselines: Digital annealing (DA), Parallel tempering (PT), Simulated annealing (SA)
- Metrics: Number of iterations, Time-to-solution (TTS), P-bit count, Solution quality (q=0.5, 0.8)
- Compute: Parallel execution assumed for replicas
