## Problem
Reward models in RLHF overlook epistemic uncertainty from limited human feedback, leading to reward overoptimization and expensive annotation costs.

## Core Claim
Model size and initialization have the most meaningful impact on uncertainty quantification performance, and most prior work could have benefited from alternative design choices.

## Key Contributions
- Unified framework RewardUQ for systematic evaluation of uncertainty quantification methods for reward models
- Comparison of common UQ methods along accuracy and calibration metrics with new ranking strategy
- Open-source Python package for framework deployment

## Method
RewardUQ trains and evaluates four uncertainty quantification architectures (ENS-MLP, ENS-LoRA, MCD-DPO, BAY-LIN) across different datasets and base models using unified evaluation procedure on RewardBench
- Components: MLP head ensemble (ENS-MLP) with K independent heads, LoRA adapter ensemble (ENS-LoRA) with parameter-efficient fine-tuning, DPO-based MC dropout (MCD-DPO) using dropout before LM head, Bayesian linear head (BAY-LIN) with Gaussian prior and Laplace approximation
- Key innovation: Unified evaluation framework enabling systematic comparison of UQ methods across standardized metrics and datasets
- Assumptions: Epistemic uncertainty arises from limited preference data; Reward and uncertainty can be separated in model architecture; Confidence bounds can be constructed using mean and standard deviation; Cross-entropy loss is appropriate for training reward models
