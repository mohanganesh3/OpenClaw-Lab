# EV028: OmniDP: Beyond-FOV Large-Workspace Humanoid Manipulation with Omnidirectional 3D Perception

## Problem
Perceptual limitations of humanoid robots in unstructured environments that constrain effective workspace, particularly narrow fields of view and self-occlusion in conventional RGB-D solutions requiring frequent repositioning.

## Core Claim
OmniDP enables robust large-workspace humanoid manipulation through omnidirectional 3D perception using an end-to-end LiDAR-driven visuomotor policy.

## Key Contributions
- Development of OmniDP, an end-to-end LiDAR-driven 3D visuomotor policy for large-workspace manipulation
- Implementation of Time-Aware Attention Pooling (TAP) mechanism for processing sparse temporal point cloud data
- Creation of a whole-body teleoperation system for efficient demonstration data collection on full-body coordination

## Method
OmniDP processes panoramic point clouds through a Time-Aware Attention Pooling mechanism that aggregates point clouds over temporal windows with attention weights conditioned on temporal offsets, producing temporally smoothed representations that condition a Diffusion Policy decoder for coordinated action sequences.
- Components: Panoramic LiDAR perception system (Livox MID-360) providing 360° horizontal and -7° to 52° vertical FOV, Time-Aware Attention Pooling (TAP) mechanism for encoding sparse 3D data with temporal dependencies, Whole-body teleoperation system using Meta Quest 3 HMD and handheld controllers for data collection
- Key innovation: Time-Aware Attention Pooling mechanism that aggregates point clouds over temporal windows and uses attention weights conditioned on temporal offsets to emphasize recent observations, providing temporally smoothed representations that improve stability for policy learning on sparse LiDAR data.
- Assumptions: Robot's arm reachability is within 1.3 meters from LiDAR sensor; Lower body can be controlled by pre-trained HOMIE algorithm during deployment; Point clouds can be represented in LiDAR's egocentric frame without extrinsic calibration

## Experimental Setup
- Datasets: Teleoperated observation-action trajectories with LiDAR point clouds and proprioceptive joint positions
- Baselines: DP [4], DP3 [5], iDP3 [6]
- Metrics: Task success rate, Collision rate
- Compute: null

## Results
Headline: OmniDP achieves 82/120 (68.3%) total success rate across all tasks, significantly outperforming baselines (DP: 18/120, DP3: 22/120, iDP3: 25/120)

- Ablations: Removing panoramic LiDAR input causes complete failure (0/20), proving necessity of 360° perception; Disabling time-aware attention pooling reduces success rate from 12/20 to 9/20, showing importance of temporal smoothing
- Improvement over baseline: OmniDP achieves 60% improvement in out-of-view tasks (12/20 vs 0/20 for all baselines) and 47.5% improvement in collision avoidance (14/20 vs 0/20)

## Critical Reading
- Stated limitations: Conventional RGB-D solutions suffer from narrow fields of view and self-occlusion; Existing approaches to expanding perception introduce mechanical complexity, calibration dependencies, and latency
- Unstated weaknesses: Focus on robotics and manipulation limits direct applicability to medical image classification; LiDAR-based 3D perception may not translate well to 2D medical imaging; Physical robot constraints differ from digital image processing environments
- Threats to validity: Domain mismatch between robotics and medical imaging; Physical environment constraints may not generalize to digital medical datasets; Different performance metrics and evaluation criteria
- Open questions: How can omnidirectional perception techniques be adapted for medical imaging?; What calibration challenges exist in medical imaging that mirror those in robotics?; Can active learning principles from robotics be applied to medical image classification?
