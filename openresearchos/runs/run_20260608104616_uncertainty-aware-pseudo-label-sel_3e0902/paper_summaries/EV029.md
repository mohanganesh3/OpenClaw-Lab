## Problem
Semi-supervised learning algorithms struggle with imbalanced data due to pseudo-label bias toward majority classes, which gets amplified during training.

## Core Claim
Existing pseudo-labeling strategies using uncalibrated confidence or heuristics are unreliable for imbalanced SSL; SEVAL addresses this through statistical analysis-based optimization of refinement and thresholds.

## Key Contributions
- SEVAL framework for imbalanced SSL using validation data
- Pseudo-label refinement via logit offset optimization
- Threshold adjustment based on precision rather than recall

## Method
SEVAL optimizes logit offsets π to minimize cross-entropy on validation data and learns thresholds τ to achieve target per-class accuracy, using a balanced validation subset separate from training data.
- Components: Pseudo-label refinement: π* = argmin π Σ H(yi, σ(zV_i - log π)), Threshold adjustment: τ* = argmin τc [A(τc) - t] if t < αc else 0
- Key innovation: Using validation data to directly optimize for precision-based thresholds and logit offsets, overcoming the unreliability of uncalibrated model confidence in imbalanced scenarios.
- Assumptions: Holdout validation data V has same class-conditional distribution as training data; Class priors are uniform in validation set (PT(Y) uniform); Access to labeled validation data separate from training/unlabeled data

## Experimental Setup
- Datasets: CIFAR-10 (n1=1500, n1=500), CIFAR-100 (n1=50), STL-10
- Baselines: DARP, LA, FixMatch, FlexMatch, FreeMatch, Mean-Teacher, MixMatch, ReMixMatch, DASO
- Metrics: sample-wise accuracy, class-wise accuracy, Gain, Correctness, test accuracy, precision, recall, ECE
