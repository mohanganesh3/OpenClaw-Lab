# EV025: CRMSP: A Semi-supervised Approach for Key Information Extraction with Class-Rebalancing and Merged Semantic Pseudo-Labeling

## Problem
Semi-supervised key information extraction faces underestimation of tail class confidence in long-tailed distributions and difficulty achieving intra-class compactness and inter-class separability of tail features.

## Core Claim
CRMSP achieves state-of-the-art performance in semi-supervised KIE by combining Class-Rebalancing Pseudo-Labeling and Merged Semantic Pseudo-Labeling modules to address long-tailed distribution challenges.

## Key Contributions
- Class-Rebalancing Pseudo-Labeling (CRP) module introduces reweighting factor to rebalance pseudo-labels, increasing attention to tail classes
- Merged Semantic Pseudo-Labeling (MSP) module clusters tail features using Merged Prototypes to improve semantic pseudo-labeling
- Overall framework achieves 3.24% F1-score improvement over state-of-the-art on CORD benchmark

## Method
CRMSP utilizes a base model with labeled/unlabeled data processing, where CRP reweights pseudo-labels using EMA-based class distribution estimates and β factor, while MSP generates merged semantic pseudo-labels via Merged Prototypes and contrastive loss.
- Components: Class-Rebalancing Pseudo-Labeling (CRP) module, Merged Semantic Pseudo-Labeling (MSP) module
- Key innovation: Reweighting factor β that reverses class distribution predictions (β = Normalize(M(˜p))) to boost tail class confidence during pseudo-label filtering
- Assumptions: Smoothness assumption: close data points in high-density regions have similar outputs; Unlabeled data distribution correlates with labeled data distribution

## Experimental Setup
- Datasets: FUNSD (149 train/50 test), CORD (800/100/100 train/val/test), CIFAR10-LT, CIFAR100-LT, STL10-LT
- Baselines: FixMatch, Dash, FlexMatch, SimMatch, FreeMatch, DARP, CReST, DASO
- Metrics: Precision, Recall, F1-score, Top-1 accuracy
- Compute: NVIDIA Tesla V100 GPU

- Ablations: RP (Reweighting Pseudo-Labels) improved F1-score from 66.56% to 67.21% on FUNSD and 84.05% to 84.76% on CORD; Lctr (Contrastive Loss) further improved F1-score to 67.75% on FUNSD and 85.36% on CORD; MP (Merged Prototypes) achieved final improvement to 71.12% on FUNSD and 89.59% on CORD; RP significantly improved F1-score for tail classes on CORD, especially for classes with smaller sample sizes
- Improvement over baseline: CRMSP improved F1-score by 5.81% compared to FixMatch on FUNSD with 5% labeled data using LayoutLMv2, improved F1-score by 4.32% compared to FixMatch on CORD with 10% labeled data using LayoutLMv3, and improved F1-score by 1.20% compared to SimMatch on FUNSD with 5% labeled data using LayoutLMv2

## Critical Reading
- Stated limitations: The paper does not explicitly state limitations in the provided abstract and highlights.
- Unstated weaknesses: No explicit discussion of uncertainty quantification in the pseudo-labeling process; Limited information on handling varying degrees of label noise in unlabeled data; No details on how the approach performs with high uncertainty scenarios; Unclear how the method handles uncertainty in feature space for tail classes
- Threats to validity: Potential sensitivity to hyperparameters related to pseudo-labeling thresholds; Limited evaluation on datasets with different uncertainty characteristics; Unclear generalizability to domains with different uncertainty patterns
- Open questions: How does the uncertainty in pseudo-labels affect the model's performance?; What is the optimal way to quantify and utilize uncertainty in the pseudo-labeling process?; How does the approach perform when the uncertainty in unlabeled data is high?; Can the method be extended to explicitly model uncertainty in the pseudo-label selection?
