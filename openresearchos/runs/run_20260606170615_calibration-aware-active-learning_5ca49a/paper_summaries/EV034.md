## Problem
The inefficiency of one-class anomaly detection models requiring separate models for each class, increasing cost and memory requirements in Industry 4.0 visual inspection systems.

## Core Claim
Multi-class unified models perform at par with one-class models for anomaly detection on MVTec AD dataset, eliminating need for separate class-wise models when object classes are significantly different.

## Key Contributions
- Experimental validation that unified multi-class models perform comparably to one-class models on MVTec AD dataset
- Deployment and analysis of three lightweight architectures on edge devices (NVIDIA Jetson Xavier NX) with quantization techniques
- Comparison of quantization-aware training (QAT) and post-training quantization (PTQ) approaches for edge deployment

## Method
The paper implements three knowledge distillation frameworks (US, RD, STFPM) to train unified multi-class models, then applies quantization techniques (PTQ and QAT) for edge deployment, analyzing performance with different precision widths.
- Components: Knowledge distillation frameworks (Uninformed Students, Reverse Distillation, STFPM), Quantization techniques (Post-Training Quantization with calibration, Quantization-Aware Training), Feature pyramid matching for enhanced anomaly detection performance
- Key innovation: Student-Teacher Feature Pyramid Matching (STFPM) which integrates hierarchical feature matching across multiple scales to enhance anomaly detection at various anomaly sizes.
- Assumptions: Object classes in the dataset are significantly different from each other; Calibration data can represent the variance encountered during testing; Teacher-student architecture can effectively transfer knowledge across multiple scales

## Experimental Setup
- Datasets: MVTec-AD
- Baselines: Uninformed Students (US), Reverse Distillation (RD), STFPM
- Metrics: AUROC, Inference Time (ms), Model Size (MB)
- Compute: CPU and NVIDIA Jetson NX device

## Results
Headline: STFPM achieves 52.14ms latency with 0.94 AUROC on Jetson (FP-32)

| Method | Dataset | Metric | Value |
|---|---|---|---|
| STFPM | MVTec-AD | Latency (ms) | 52.14 |
| STFPM | MVTec-AD | AUROC | 0.94 |
| RD | MVTec-AD | AUROC | 0.96 |
| RD | MVTec-AD | Latency (ms) | 166.60 |
| US | MVTec-AD | Latency (ms) | 18274 |
| STFPM | MVTec-AD | Model Size (MB) | 88 |
| RD | MVTec-AD | Model Size (MB) | 644.2 |
| US | MVTec-AD | Model Size (MB) | 26.8 |

- Ablations: Quantization precision (FP-32 vs FP-16 vs INT-8); Calibration strategy (training data vs random normal); Framework (PyTorch vs TensorRT)
- Improvement over baseline: STFPM is 5x faster than RD on Jetson (52.14ms vs 166.60ms) with only 0.02 AUROC difference; Random normal calibration improves RD AUROC by 0.25 (0.50→0.75) and STFPM by 0.08 (0.75→0.83)
