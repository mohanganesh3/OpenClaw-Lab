# EV007: Self-Refine: Iterative Refinement with Self-Feedback

Year: 2023 | Relation: tangential | Repro: medium | Source: pymupdf_full (54p, 21 tables)
URL: https://arxiv.org/abs/2303.17651 | Code: no

## Problem
Large language models often fall short in addressing intricate requirements and multifaceted objectives, producing intelligible initial outputs that could benefit from iterative refinement.

## Core Claim
SELF-REFINE improves LLM outputs through iterative self-feedback and refinement without requiring additional training data, supervised training, or reinforcement learning.

## Key Contributions
- Introduction of SELF-REFINE, an approach for iterative refinement using self-feedback
- Demonstration that state-of-the-art LLMs like GPT-4 can be further improved at test-time using this simple approach
- Evaluation across 7 diverse tasks showing ~20% absolute improvement in task performance

## Method
SELF-REFINE generates an initial output using an LLM, then the same LLM provides feedback for its output and uses it to refine itself iteratively across multiple steps.
- Components: Initial output generation using an LLM, Self-feedback generation by the same LLM, Iterative refinement using the feedback
- Key innovation: Using the same LLM as both the generator and the feedback provider, eliminating the need for additional training data or models.
- Assumptions: The LLM can provide meaningful feedback on its own outputs; Iterative refinement will lead to improvement; State-of-the-art LLMs can benefit from this approach

## Experimental Setup
- Datasets: CommonGen-Hard, Acronym Generation dataset (250 acronyms), CodeNet (300 examples)
- Baselines: Base GPT-3.5, Base ChatGPT, Base GPT-4
- Metrics: Preference evaluation, Solve rate, Percentage of programs optimized, Coverage percentage, Meaningful Variable Ratio, Comment Per Line, Function Units
- Compute: Not specified

## Results
Headline: GPT-4: 93.1% in Math Reasoning (vs 92.9% base)

- Ablations: Robustness to bad feedback (60% of cases); Ignoring good feedback (25% of cases); Introducing new problems (20% of cases)
- Improvement over baseline: GPT-4: +0.2% (Math Reasoning), GPT-3.5: +21.6% (Sentiment Reversal), ChatGPT: +19.8% (Dialogue Response)

## Critical Reading
- Stated limitations: The paper doesn't explicitly state limitations in the abstract; The method focuses on text generation rather than image classification; No mention of computational efficiency or scalability
- Unstated weaknesses: No discussion of how self-feedback reliability is ensured; No evaluation on specialized domains like medical imaging; Potential for reinforcing biases through iterative refinement; No mention of handling uncertainty or confidence in outputs
- Threats to validity: Self-feedback mechanism may not be reliable for critical applications; No evidence of generalization to non-text domains; Potential overfitting to specific tasks evaluated; No discussion of handling rare medical conditions
- Open questions: How would self-refine work for image classification tasks?; Can the method provide calibrated confidence scores for medical images?; How does iterative refinement affect computational efficiency for large medical datasets?; Can this approach be integrated with active learning for medical imaging?
