# EV010: SWE-agent: Agent-Computer Interfaces Enable Automated Software Engineering

## Problem
Language model agents lack specialized interfaces for complex software engineering tasks, limiting their performance compared to human engineers using tools like IDEs.

## Core Claim
Custom agent-computer interfaces significantly enhance LM agents' ability to autonomously perform software engineering tasks.

## Key Contributions
- Introduction of SWE-agent system with custom agent-computer interface (ACI)
- Achieving state-of-the-art performance on SWE-bench (12.5% pass@1) and HumanEvalFix (87.7%)
- Demonstrating that interface design critically impacts agent behavior and performance

## Method
SWE-agent enables LMs to interact with computers through a custom interface that translates LM actions into filesystem operations, file edits, and program executions with structured feedback.
- Components: LM Agent core, Agent-Computer Interface (ACI) with LM-friendly commands, Computer environment (filesystem, terminal)
- Key innovation: The ACI's LM-friendly command structure and environment feedback format, which bridges the gap between LM reasoning and computer operations by providing structured, interpretable interactions.
- Assumptions: LMs can effectively follow structured command interfaces; Repository navigation and file editing are primary bottlenecks in software engineering tasks; Execution feedback is critical for iterative problem-solving

## Experimental Setup
- Datasets: SWE-bench Lite dataset, SWE-bench test set (286 trajectories for GPT-4, 35 trajectories for Claude 3 Opus)
- Baselines: SWE-agent with GPT-4, SWE-agent with Claude 3 Opus
- Metrics: Resolution rate by turn, Total steps, Cost in USD, Action pattern frequency
- Compute: Not specified
