## Problem
Language model agents struggle with complex software engineering tasks when using standard interfaces, unlike human engineers who benefit from sophisticated tools like IDEs.

## Core Claim
A custom agent-computer interface (ACI) specifically designed for language model agents significantly enhances their ability to autonomously solve software engineering tasks.

## Key Contributions
- Introduction of SWE-agent system with custom ACI for LM agents
- State-of-the-art performance on SWE-bench (12.5% pass@1) and HumanEvalFix (87.7%) benchmarks

## Method
SWE-agent uses a language model agent interacting with a computer through a custom ACI that includes LM-friendly commands for navigation, file operations, and execution, with structured feedback from the environment.
- Components: Filesystem navigation and viewing capabilities, Search and edit functions with line-level precision, Terminal integration for program execution, LM-friendly command structure and environment feedback
- Key innovation: The agent-computer interface (ACI) design that bridges the gap between LM capabilities and software engineering requirements through carefully crafted commands and feedback mechanisms.
- Assumptions: Language model agents can effectively utilize structured interfaces; Interface design significantly impacts agent performance; LM agents benefit from specialized tools similar to human engineers

## Experimental Setup
- Datasets: SWE-bench Lite dataset, SWE-bench test set (286 trajectories), SWE-bench test set Lite split (35 trajectories)
- Baselines: SWE-agent with GPT-4, SWE-agent with Claude 3 Opus
