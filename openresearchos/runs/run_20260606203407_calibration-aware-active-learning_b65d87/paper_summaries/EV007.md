# EV007: Self-Refine: Iterative Refinement with Self-Feedback

Year: 2023 | Relation: tangential | Repro: low (without implementation details in abstract) | Source: pymupdf_full (54p, 21 tables)
URL: https://arxiv.org/abs/2303.17651 | Code: no

## Problem
Large language models often generate suboptimal outputs on first attempts, especially for tasks with multifaceted objectives or hard-to-define goals, requiring iterative refinement to achieve desired quality.

## Core Claim
SELF-REFINE demonstrates that state-of-the-art LLMs can be significantly improved through iterative self-feedback and refinement at test-time without additional training or supervision.

## Key Contributions
- A novel approach for iterative refinement using self-feedback within a single LLM
- Empirical evaluation across 7 diverse tasks showing ~20% absolute improvement in task performance
- Demonstration that no supervised training data, additional training, or reinforcement learning is required

## Method
SELF-REFINE generates an initial output using an LLM, then iteratively refines it by having the same LLM provide feedback on its output and use this feedback to produce improved versions.
- Components: Initial output generation, Self-feedback generation, Iterative refinement loop
- Key innovation: The self-feedback mechanism where the same LLM evaluates its own output and generates improvements, creating a closed-loop refinement process without external supervision.
- Assumptions: LLMs can provide meaningful feedback about their own outputs; Iterative refinement will consistently improve output quality; The same LLM can effectively perform generation, feedback, and refinement tasks

## Experimental Setup
- Datasets: CommonGen-Hard (extension of CommonGen dataset), Acronym dataset (250 acronyms, manually pruned), CodeNet (competitive programming dataset, 300 examples), Dialogue Response Generation dataset, Sentiment Reversal dataset, Code Optimization dataset, Math Reasoning dataset, Acronym Generation dataset, Constrained Generation dataset
- Baselines: Base GPT-3.5, Base ChatGPT, Base GPT-4
- Metrics: Solve rate (Math Reasoning), Percentage of programs optimized (Code Optimization), GPT-4-based preference evaluation (Sentiment Reversal, Dialogue Response, Acronym Gen), Coverage percentage (Constrained Gen), Meaningful Variable Ratio, Comment Per Line, Function Units, Wilson confidence interval (99% CI)
- Compute: Not explicitly stated

## Results
Headline: GPT-4 + SELF-REFINE achieves 93.1% on Math Reasoning (vs 92.9% base), with 36.2% on Sentiment Reversal (vs 3.8% base)

| Method | Dataset | Metric | Value |
|---|---|---|---|
| GPT-4 + SELF-REFINE | Math Reasoning | solve rate | 93.1 ± 2.03 |
| GPT-4 Base | Math Reasoning | solve rate | 92.9 ± 2.05 |
| GPT-4 + SELF-REFINE | Sentiment Reversal | preference eval | 36.2 ± 3.82* |
| GPT-4 Base | Sentiment Reversal | preference eval | 3.8 ± 1.28 |
| SELF-REFINE (T=0.7) | Code Readability | Meaningful Variable Ratio | 0.700 |
| Human Annotator | Code Readability | Meaningful Variable Ratio | 0.653 |

- Ablations: Temperature settings for critique (0.0 vs 0.7); Number of refinement iterations (N=5); Base model selection (GPT-3.5/ChatGPT/GPT-4)
- Improvement over baseline: Statistically significant improvements (* marked) across 10/14 tasks: GPT-4 gains significant in 7/7 tasks, ChatGPT in 4/7, GPT-3.5 in 3/7. Code Readability: SELF-REFINE (T=0.7) exceeds human performance on Meaningful Variable Ratio (0.700 vs 0.653).

## Critical Reading
- Stated limitations: The paper doesn't explicitly mention limitations in the abstract
- Unstated weaknesses: The approach focuses on text generation rather than image classification; Unclear how well the self-feedback mechanism would work for visual data; Potential computational expense for high-resolution medical images; Risk of error accumulation in iterative refinement for critical medical applications
- Threats to validity: Applicability to medical image classification is uncertain; Medical image classification requires high precision which might not be guaranteed; Iterative refinement could introduce biases or errors
- Open questions: How can self-refinement be adapted for image classification?; Can the feedback mechanism be effectively applied to visual data?; How does this compare to traditional active learning for medical images?; What are the computational requirements for medical image applications?
