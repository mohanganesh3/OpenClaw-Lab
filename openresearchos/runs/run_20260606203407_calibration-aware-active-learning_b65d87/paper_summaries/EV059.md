## Problem
Fundamental limitations of reinforcement learning from human feedback (RLHF) under misspecification, specifically how systematic bias on a subset of contexts leads to a persistent performance gap that cannot be overcome with bounded query budgets.

## Core Claim
The performance gap always wins under misspecification—any learner using a bounded number of queries will face a gap scaling exponentially with queries, unless a calibration oracle identifies misspecified contexts.

## Key Contributions
- Lower bound on performance gap for any learner: ≥γ/4 * exp(-nακ(ε)) with κ(ε) = 4ε atanh(2ε)
- Upper bound with calibration oracle: Achieves gap ≤η with Q ≤ (1/(2αε²)) log(γ/η) queries
- Tilted calculus framework to analyze optimization drift and distribution shift sensitivity

## Method
Uses Le Cam's two-point method with transcript-level KL decomposition for lower bounds, and majority testing on flagged contexts for upper bounds, combined with tilted calculus to analyze optimization dynamics.
- Components: Le Cam's two-point method and KL divergence for lower bound derivation, Majority test on flagged contexts for query-efficient gap reduction, Tilted sampling law qλ and covariance analysis for optimization drift
- Key innovation: Identification that hard mass α and bias ε govern sample complexity, and that calibration oracles enable logarithmic query scaling via context routing—transforming exponential to polynomial complexity.
- Assumptions: Contexts drawn i.i.d. from distribution µ over X; Existence of measurable Xhard with mass α where feedback is biased; Pairwise label favors better action with prob 1/2 + wε on Xhard (ε ∈ (0, 1/2)); Separation parameter γ > 0 defined as expected reward difference on Xhard; For upper bound: Access to calibration oracle h(x) = 1{x ∈Xhard}
