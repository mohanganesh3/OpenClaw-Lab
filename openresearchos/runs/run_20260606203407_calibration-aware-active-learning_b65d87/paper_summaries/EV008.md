# EV008: Voyager: An Open-Ended Embodied Agent with Large Language Models

Year: 2023 | Relation: tangential | Repro: medium | Source: pymupdf_full (42p, 13 tables)
URL: https://arxiv.org/abs/2305.16291 | Code: https://voyager.minedojo.org

## Problem
Building generally capable embodied agents that can continuously explore, plan, and develop new skills in open-ended worlds without human intervention

## Core Claim
VOYAGER demonstrates that a three-component system combining an automatic curriculum, skill library, and iterative prompting mechanism can achieve open-ended lifelong learning in Minecraft

## Key Contributions
- First LLM-powered embodied lifelong learning agent in Minecraft
- Three-component architecture enabling continuous exploration and skill acquisition
- Demonstrates superior performance (3.3× items, 2.3× distance, 15.3× speed) over baselines

## Method
VOYAGER employs GPT-4 via blackbox queries to generate executable code for Minecraft actions, organized into a three-component system: an automatic curriculum for exploration, a skill library for storing behaviors, and an iterative prompting mechanism that refines code using environment feedback, execution errors, and self-verification
- Components: Automatic curriculum that maximizes exploration through curiosity-driven task selection, Ever-growing skill library of executable code for storing and retrieving complex behaviors, Iterative prompting mechanism incorporating environment feedback, execution errors, and self-verification
- Key innovation: The iterative prompting mechanism that integrates environment feedback, execution errors, and self-verification to continuously improve generated code without model fine-tuning
- Assumptions: GPT-4's coding capabilities are sufficient for generating correct Minecraft actions; Self-verification can accurately assess task completion; Automatic curriculum can propose achievable tasks
