## Problem
Conformal outlier detection methods become conservative and lose power when trained on contaminated reference data containing outliers, which is a realistic practical scenario.

## Core Claim
Standard conformal methods inherently control type-I error conservatively under non-adversarial contamination, but this conservativeness can be mitigated through an active data-cleaning framework that selectively removes suspected outliers.

## Key Contributions
- Proves that calibrating on contaminated reference data yields conservative type-I error control in non-adversarial settings
- Proposes an active data-cleaning framework leveraging limited labeling budget and outlier detection models to selectively annotate and remove suspected outliers, enhancing power without sacrificing validity

## Method
The framework uses an outlier detection model to identify suspicious points in the contaminated reference set, then leverages a limited labeling budget to selectively annotate and remove only these points before conformal calibration.
- Components: Conformal outlier detection framework, Active data-cleaning mechanism with labeling budget, Outlier detection model for identifying suspicious points
- Key innovation: Selective cleaning of only suspected outliers (via active labeling) rather than full cleaning or no cleaning, preserving the conservative validity of conformal methods while improving power.
- Assumptions: Contamination is non-adversarial (small fraction of outliers); Inliers in the reference set are i.i.d. from the true distribution; Scores have no ties (achievable via negligible random noise); Exchangeability holds for inlier scores

## Experimental Setup
- Datasets: six visual datasets, Texture dataset (outliers) and CIFAR-10 dataset (inliers), SVHN dataset (outliers) and CIFAR-10 dataset (inliers)
- Baselines: Standard, Oracle (infeasible), Naive-Trim (invalid), Small-Clean, Label-Trim
- Metrics: Power, Type-I Error

## Critical Reading
- Stated limitations: The conservativeness of conformal methods under contamination typically results in a loss of power.
- Unstated weaknesses: The abstract doesn't specify the exact type of contamination considered; No performance metrics for the proposed method; No discussion of computational complexity; No mention of scalability to large datasets; Limited information about the experimental setup
- Threats to validity: The paper focuses on 'realistic, non-adversarial settings' but doesn't define what constitutes realistic contamination; The conservativeness of conformal methods under contamination might not be sufficient for critical medical applications; The proposed active cleaning framework's effectiveness with very high contamination rates is unclear
- Open questions: How does the proposed active cleaning framework perform with different types of contamination?; What is the optimal labeling budget for the active cleaning process?; How does this method scale to high-dimensional medical images?; Can the framework be extended to handle multiple types of outliers?
